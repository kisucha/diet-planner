# 요구사항 분석 문서

이 문서는 자동 식단 생성 웹사이트의 요구사항을 분석하고 정의합니다.

**최종 업데이트**: 2026-03-22 (v4.0 전면 개정 — 음식 분류 시스템, 식단 구성 리스트, 특별일/주말 제안 반영)

---

## 1. 프로젝트 개요

자동 식단 생성 웹사이트는 사용자가 큐레이션한 음식 리스트를 기반으로 일일 식단을 자동으로 생성해주는 시스템입니다. 인터넷에서 레시피와 재료를 자동 수집하고, 음식을 종류별로 분류(메인요리/반찬/야식)하여 체계적인 식단을 구성합니다. 메인재료/양념을 구분하여 실제 재료비를 산출하며, 명절/기념일 특별 음식 제안과 주말 특식 제안 기능을 포함합니다.

---

## 2. 핵심 설계 원칙 (v4.0)

| 원칙 | 설명 |
|------|------|
| **2단계 음식 리스트** | 전체 음식 DB(recipes)와 식단 구성 리스트(meal_list)는 별개. 식단은 리스트에서만 생성 |
| **세부 음식 분류** | 한국요리: 요리/탕/찌개/국으로 세분화. 외국요리: 단일 분류. 면요리: 국(guk)으로 분류 |
| **반찬 4개 자동 추천** | 식단 생성 시 반찬 리스트에서 4개 자동 추천, 수정 가능 |
| **식사 시간 지정** | 메인요리는 리스트 등록 시 조식/중식/석식 지정 (복수 선택 가능) |
| **특별일 제안** | 추석/설날/크리스마스/추수감사절에 해당 음식 별도 제안 (강제 적용 아님) |
| **주말 특식 제안** | 주말(토/일)에는 일반 식단 + 시스템 특식 목록에서 별도 제안 |

## 3. 화면별 기능 요구사항

### 3.1 기본화면 (사용자 뷰) — `/`

#### 3.1.1 화면 크기
- **기준**: 아이패드 가로(Landscape) 모드 기준
- **해상도**: 1024×768px (iPad mini/Air), 최대 1366px (iPad Pro) 대응
- 세로 모드나 모바일은 부수적으로 지원 (가로 우선)

#### 3.1.2 오늘의 식단 표시
- 기본화면 진입 시 **오늘 날짜**의 식단 자동 표시
- **5가지 구성**: 조식(아침) / 중식(점심) / 석식(저녁) / 야식 / **반찬 4개**
- 각 식사란에 해당 음식명 표시
- 반찬은 4개를 가로로 나열하여 표시
- 식단이 미작성된 경우: "오늘은 식단이 작성되지 않았습니다" 안내 메시지 표시

#### 3.1.3 음식 상세 보기
- 식사 항목(조식/중식/석식/야식) 클릭 → 해당 음식의 **사진 + 조리 방법** 표시
- **"자세히보기" 또는 "디테일" 버튼** 클릭 → **재료 목록** 펼쳐 표시
  - 메인재료와 양념을 구분하여 표시
  - 각 재료의 필요 용량 함께 표시

#### 3.1.4 주말 특식 배너 (주말만 표시)
- 토요일/일요일에만 화면 하단에 특식 제안 배너 표시
- 시스템 특식 목록(짜장면, 탕수육, 피자 등)에서 3~5개 제안
- 사용자가 선택 시 해당 식사를 특식으로 교체 가능

#### 3.1.5 특별한 날 배너 (해당일만 표시)
- 추석/설날/크리스마스/추수감사절 해당 날짜에만 배너 표시
- "오늘은 추석입니다! 명절 음식을 추천해 드립니다" 형식
- 해당 명절 음식 목록 표시 (송편, 갈비찜 등)
- 사용자가 선택 시 식단에 적용 가능

### 3.2 관리화면 (Admin 뷰)

#### 3.2.1 식단 자동 생성 (`/admin/meal-plan`) — 핵심 기능
- **가장 중요한 기능**: 등록된 식단 구성 리스트 기반으로 식단 자동 생성
- 날짜 또는 기간 선택 후 [자동 생성] 버튼으로 식단 생성
- 생성 시 규칙:
  - 메인요리: 각 식사 유형별(조/중/석식)로 허용된 음식에서 랜덤 1개 배정
  - 반찬: 반찬 리스트에서 4개 랜덤 배정
  - 야식: 야식/별식 리스트에서 랜덤 1개 배정 (없으면 미지정)
  - 최근 3일 이내 동일 음식 반복 배정 방지
- 주말이면 특식 제안 자동 표시
- 특별한 날이면 해당 명절 음식 제안 자동 표시

#### 3.2.2 식단 조회 및 수정
- 달력 형태로 월별 식단 현황 표시
- 식단이 있는 날짜: 칼·주방도구 아이콘(🔪) 마커 표시
- 식단이 없는 날짜: 마커 없음
- 날짜 클릭 → 해당 날짜의 식단 상세 + 수동 수정 가능
- 각 식사(조/중/석식/야식)별 메인요리 개별 변경 가능
- 반찬 4개 각각 개별 변경 가능

#### 3.2.3 재료비 자동 산정
- 식단이 생성되면 **메인재료 기준**으로 총 재료비 자동 산정
- 계산 대상: 메인요리(조/중/석/야식) + 반찬 4개의 메인재료 합산
- **양념류는 재료비 산정에서 제외**
- 각 재료별 비용 상세 내역 표시
- 미등록 재료가 있을 경우 해당 재료 "가격 미등록" 표시

#### 3.2.4 미작성 식단 처리
- 기본화면에서 식단이 없는 날짜 접근 시 → 안내 메시지 표시
- 관리화면에서 미작성 날짜 일괄 확인 가능

---

### 3.3 식단 구성 리스트 관리 화면 (`/admin/meal-list`) [신규]

#### 3.3.1 목적
- 전체 음식 DB에서 실제 식단 생성에 사용할 음식들을 선별하여 관리
- 이 리스트에 있는 음식에서만 식단이 생성됨
- 언제든지 추가/제거/비활성화 가능

#### 3.3.2 탭 구성
- **메인요리 탭**: 조/중/석식에 배정되는 메인 요리 목록
- **반찬 탭**: 4개 반찬으로 배정되는 반찬 목록
- **야식/별식 탭**: 야식으로 배정되는 음식 목록

#### 3.3.3 메인요리 리스트 관리
- 분류 필터: 전체/한식-요리/한식-탕/한식-찌개/한식-국/외국요리
- 목록 표시: 사진, 음식명, 분류, 추천 식사 시간(조/중/석식 체크박스), 액션
- **추천 식사 시간 설정 (필수)**: 메인요리 등록 시 조식/중식/석식 중 1개 이상 반드시 선택
  - 예: 김치찌개 → [중식✓] [석식✓] 선택
  - 선택된 식사 시간에만 해당 음식이 추천됨
- 식사 시간 수정 가능

#### 3.3.4 전체 DB에서 리스트에 추가
- [전체 음식 DB에서 추가] 버튼 클릭 → 팝업/모달 창
- 검색 + 분류 필터로 원하는 음식 선택
- 선택 후 [리스트에 추가] → 메인요리의 경우 식사 시간 지정 단계 추가
- 이미 리스트에 있는 음식은 선택 불가(중복 방지)

#### 3.3.5 반찬 리스트 관리
- 반찬 목록 표시 및 추가/제거
- 식사 시간 선택 불필요 (모든 식단의 반찬으로 사용됨)

### 3.4 음식 등록 화면 (`/admin/foods`)

#### 3.4.1 음식 분류 등록 (신규)
- 음식 등록 시 **분류 지정 필수**:
  - 대분류: 메인요리 / 반찬 / 야식별식
  - 메인요리 세부 분류 (한국요리):
    - 요리 (볶음/구이/튀김/무침)
    - 탕 (탕/찜)
    - 찌개 (찌개/전골)
    - 국 (국/면요리 — 라면, 칼국수 포함)
  - 외국요리: 분류 없이 "외국요리" 단일 선택
- 면 요리(라면, 칼국수, 냉면 등)는 "국" 분류 안내 문구 표시

#### 3.4.2 인터넷 자동 레시피 수집 (핵심)
- 음식명 입력 → 인터넷에서 관련 레시피 자동 검색
- 수집 대상:
  - **레시피 (조리 방법)**: 단계별 조리 순서
  - **재료 목록**: 재료명 + 필요 용량
  - **음식 사진**: 대표 이미지 URL
- 한국 음식 → 만개의 레시피(10000recipe.com) 우선 사용
- 영어 음식 → Spoonacular API 사용

#### 3.4.3 메인재료 vs 양념 자동 구분 (핵심)
- 수집된 재료를 **메인재료**와 **양념**으로 자동 분류
  - 메인재료: 김치, 돼지고기, 두부, 대파 등 → 가격 산정 대상
  - 양념: 소금, 간장, 참기름, 고추가루 등 → 가격 산정 제외
- 자동 분류 후 사용자가 수동으로 수정 가능
- 재료를 양쪽 열(메인재료/양념)에서 드래그 또는 버튼으로 이동

#### 3.4.4 재료 용량 수집
- 가능한 경우 재료의 **필요 용량** 함께 수집 (예: 김치 200g, 돼지고기 150g)
- 단위: g, ml, 개, 스푼 등 다양한 단위 지원
- 용량 수동 수정 가능

#### 3.4.5 대량 스크래핑
- 특정 사이트(만개의 레시피 등)에서 **대량 레시피 일괄 수집** 가능
- 카테고리별 수집 (한식, 양식 등)
- 수집 진행 상황 실시간 표시 (SSE 기반)
- 이미 등록된 레시피는 중복 수집 방지 (source_url 기준)

#### 3.4.6 외부 API 키 관리 (`/admin/settings`)
- Spoonacular API 키 등록/수정
- 네이버 검색 API Client ID / Client Secret 등록
- API 연결 상태 테스트 기능

### 3.5 재료별 가격 등록 화면 (`/admin/ingredients`)

#### 3.5.1 재료 가격 등록
- 재료명, 구매 기준량(용량), 단위, 가격 입력
- 예: 돼지고기 / 500 / g / 9,800원
- 단위당 가격 자동 계산 표시 (9,800 ÷ 500 = 19.6원/g)

#### 3.5.2 재료비 산정 활용
- 등록된 가격 정보를 바탕으로 식단 생성 시 재료비 자동 산정
- 계산 공식: 필요 용량 × 단위당 가격 = 해당 재료 비용
- 메인요리 + 반찬 4개 메인재료만 합산 → 식단 총 재료비

#### 3.5.3 가격 관리
- 등록된 재료 가격 수정/삭제 가능
- 재료명으로 검색 기능
- 가격 미등록 재료 목록 별도 표시 (식단 생성 후 산정 불가 재료)

---

### 3.6 특별한 날 관리 화면 (`/admin/holiday`) [신규]

#### 3.6.1 특별한 날 날짜 등록
- 지원 명절/기념일: 추석, 설날, 크리스마스, 추수감사절
- 추석/설날: 매년 날짜 수동 등록 (음력 계산 불가로 수동 입력)
- 크리스마스/추수감사절: 자동 계산이지만 테이블에서 함께 관리
- 날짜 범위 설정 가능 (추석: 3일 연휴 등)

#### 3.6.2 특별한 날 추천 음식 관리
- 각 명절별 추천 음식 목록 관리
- 추천 음식 추가/수정/삭제
- 추천 음식이 전체 DB에 있으면 자동 연결, 없으면 이름만 표시

#### 3.6.3 주말 특식 목록 관리
- 시스템 기본 특식 목록 표시 (짜장면, 탕수육, 피자 등)
- 특식 추가/수정/삭제/비활성화

## 4. 특별한 날 제안 시스템

### 4.1 지원하는 특별한 날

| 명절/기념일 | 날짜 기준 | 대표 음식 |
|------------|----------|----------|
| **추석** | 음력 8월 15일 (±2일) | 송편, 갈비찜, 잡채, 전, 나물 |
| **설날** | 음력 1월 1일 (±2일) | 떡국, 갈비찜, 잡채, 전류 |
| **크리스마스** | 12월 25일 (±1일) | 로스트 치킨, 스테이크, 케이크 |
| **추수감사절** | 11월 넷째 목요일 | 로스트 터키, 호박파이, 매쉬드 포테이토 |

### 4.2 제안 방식
- 해당 날짜 식단 생성 시 자동으로 별도 섹션에 표시
- 기존 식단은 그대로 유지하면서 "제안" 형태로만 보여줌
- 사용자가 [특별 음식으로 교체] 버튼으로 선택 시 식단에 반영

## 5. 주말 특식 제안 시스템

### 5.1 주말 특식 개념
- 평소에는 자주 먹지 않지만 주말에 먹기 좋은 음식
- 시스템이 기본 특식 목록 보유 (사용자 추가/수정 가능)
- meal_list에 없어도 제안 가능

### 5.2 주말 식단 생성 방식
- meal_list의 일반 식단 추천과 동시에 시스템 특식 목록에서 3~5개 별도 제안
- "오늘의 특식 추천" 섹션으로 분리하여 표시
- 사용자가 선택 시 해당 식사를 특식으로 교체

---

## 6. 비기능 요구사항

### 6.1 성능
- 기본화면 로딩: 2초 이내
- 인터넷 레시피 검색: 10초 이내 (외부 API 의존)
- 대량 스크래핑: 백그라운드 처리 (사용자 대기 최소화, SSE로 진행상황 표시)

### 6.2 보안
- 외부 API 키: 서버 환경 변수(.env)에만 저장, 클라이언트에 미노출
- DB 접근: 파라미터 바인딩으로 SQL 인젝션 방지
- 스크래핑: 출처 URL 저장, 저작권 표시

### 6.3 사용성
- 아이패드 가로 모드 최우선 최적화
- 터치 친화적 UI (버튼 크기 최소 44×44px)
- 직관적인 메인재료/양념 구분 편집

## 8. 데이터 요구사항

### 8.1 핵심 데이터 엔티티

| 엔티티 | 주요 속성 |
|--------|----------|
| 전체 음식 DB(recipes) | 음식명, 분류(대/세부), 원산지, 조리법, 이미지, 출처 |
| 레시피재료(recipe_ingredients) | 재료명, 용량, 단위, **메인/양념 구분** |
| 식단 구성 리스트(meal_list) | 레시피 ID, 조식/중식/석식 허용 여부, 활성 여부 |
| 식단(meal_plans) | 날짜, 조식/중식/석식/야식 ID, 총비용, 특별일 여부 |
| 식단 반찬(meal_plan_sides) | 식단 ID, 반찬 레시피 ID, 순서 |
| 재료가격(ingredient_prices) | 재료명, 구매량, 단위, 가격, 단위당가격 |
| 특별한 날(holiday_calendar) | 연도, 명절 키, 시작일, 종료일 |
| 특별일 추천(holiday_suggestions) | 명절 키, 음식명, 레시피 연결 |
| 주말 특식(weekend_specials) | 음식명, 레시피 연결, 활성 여부 |
| API키(external_api_keys) | 서비스명, API키 |

### 8.2 핵심 데이터 흐름

```
음식명 입력
    → 인터넷 레시피 수집 (스크래핑/API)
    → 음식 분류 지정 (메인요리/반찬/야식 + 세부분류)
    → 메인재료/양념 자동 분류
    → DB 저장 (recipes + recipe_ingredients)
    ↓
식단 구성 리스트 관리
    → 전체 DB에서 원하는 음식 선택
    → 메인요리: 식사 시간(조/중/석식) 지정
    → meal_list 저장
    ↓
식단 자동 생성
    → 날짜 성격 분석 (평일/주말/특별일)
    → meal_list에서 식사 유형별 랜덤 메인요리 배정
    → meal_list 반찬에서 4개 랜덤 배정
    → 주말이면 weekend_specials 제안 생성
    → 특별일이면 holiday_suggestions 제안 생성
    → DB 저장 (meal_plans + meal_plan_sides)
    → 재료비 자동 산정 → total_cost 저장
    ↓
기본화면
    → 오늘 날짜 meal_plans + meal_plan_sides 조회
    → 각 식사의 recipes 조회
    → 음식 사진 + 조리법 + 재료 표시
    → 필요시 주말 특식 배너 / 특별일 배너 표시
```

## 9. 제약사항

### 9.1 기술 제약
- MariaDB 필수 사용
- Node.js + Express 기반 웹 애플리케이션
- EJS 템플릿 엔진
- axios + cheerio: package.json에 등록 필요 (현재 누락)

### 9.2 운영 제약
- Spoonacular API: 무료 플랜 하루 150포인트 제한
- 만개의 레시피 스크래핑: 요청 간 딜레이 필수 (서버 부하 방지)
- 코딩 시작 전 계획 검토 필수
- 추석/설날: 음력 기반으로 매년 날짜 수동 등록 필요

### 9.3 식단 생성 제약
- 식단 생성은 반드시 meal_list에 등록된 음식에서만 이루어짐
- 메인요리는 리스트에 등록 시 식사 시간 1개 이상 반드시 지정
- 반찬은 최소 4개 이상 리스트에 있어야 식단 생성 가능
- 최근 3일 이내 동일 음식 중복 방지 (리스트 음식 수가 3개 이하면 예외 허용)
