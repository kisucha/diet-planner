# 시스템 아키텍처 설계 문서

이 문서는 자동 식단 생성 웹사이트의 시스템 아키텍처를 설계하고 정의합니다.

## 1. 아키텍처 개요

시스템은 전통적인 MVC(Model-View-Controller) 패턴을 기반으로 설계되며, 서비스 계층을 추가하여 비즈니스 로직을 효과적으로 분리합니다. 클라이언트는 웹 브라우저를 통해 서버에 요청을 보내고, 서버는 데이터베이스와 상호작용하여 필요한 정보를 제공합니다.

## 2. 시스템 구성도

```
┌─────────────────────────────────────────────────────────────┐
│                     클라이언트 (브라우저)                   │
└─────────────────────────┬───────────────────────────────────┘
                          │ HTTP 요청/응답
┌─────────────────────────▼───────────────────────────────────┐
│                        웹 서버                              │
│  ┌───────────────┐    ┌───────────────┐    ┌──────────────┐ │
│  │   컨트롤러    │◄──►│   서비스      │◄──►│   모델       │ │
│  │               │    │               │    │              │ │
│  └───────────────┘    └───────────────┘    └──────────────┘ │
└─────────────────────────┬───────────────────────────────────┘
                          │
┌─────────────────────────▼───────────────────────────────────┐
│                      데이터베이스 (MariaDB)                 │
└─────────────────────────────────────────────────────────────┘
```

## 3. 계층별 구성요소

### 3.1 프레젠테이션 계층 (Presentation Layer)

#### 3.1.1 뷰 (Views)
- EJS 템플릿 엔진을 사용한 동적 HTML 생성
- 달력 기반 사용자 인터페이스
- 식단 표시 및 레시피 등록 폼

#### 3.1.2 정적 리소스 (Static Resources)
- CSS: 스타일 시트
- JavaScript: 클라이언트 사이드 상호작용
- Images: 아이콘 및 기타 이미지 리소스

### 3.2 애플리케이션 계층 (Application Layer)

#### 3.2.1 컨트롤러 (Controllers)
- HTTP 요청 처리 및 응답 생성
- 입력 데이터 유효성 검사
- 서비스 계층 호출

#### 3.2.2 서비스 (Services)
- 핵심 비즈니스 로직 처리
- 레시피 관리 및 카테고리 분류
- 식단 생성 알고리즘
- 재료 비용 계산
- 인터넷 학습 기능 통합

#### 3.2.3 라우트 (Routes)
- URL 엔드포인트 정의
- 요청 라우팅

### 3.3 데이터 계층 (Data Layer)

#### 3.3.1 모델 (Models)
- 데이터베이스 테이블과의 매핑
- CRUD(Create, Read, Update, Delete) 연산
- 데이터 유효성 검사

#### 3.3.2 데이터베이스 (Database)
- MariaDB를 사용한 데이터 저장
- 레시피, 식단, 재료 가격 정보 관리

## 4. 데이터 흐름

### 4.1 레시피 등록 흐름
1. 사용자가 레시피 등록 폼 작성
2. 컨트롤러에서 데이터 수신 및 유효성 검사
3. 서비스 계층에서 비즈니스 로직 처리
4. 모델을 통해 데이터베이스에 저장

### 4.2 식단 생성 흐름
1. 사용자가 기간 선택 및 식단 요청
2. 컨트롤러에서 요청 수신
3. 서비스 계층에서 식단 생성 알고리즘 실행
4. 필요한 레시피 데이터 조회
5. 생성된 식단을 데이터베이스에 저장
6. 결과를 뷰로 반환

### 4.3 달력 표시 흐름
1. 사용자가 달력 페이지 요청
2. 컨트롤러에서 요청 처리
3. 서비스 계층에서 식단 데이터 조회
4. 뷰에서 달력 형식으로 렌더링
5. 계획된 날짜는 특수 마커로 표시

## 5. 보안 고려사항

### 5.1 입력 검증
- 모든 사용자 입력에 대한 유효성 검사
- SQL 인젝션 방지를 위한 파라미터 바인딩

### 5.2 접근 제어
- 데이터베이스 접근 권한 최소화
- 민감한 설정 정보 환경 변수로 분리

## 6. 확장성 및 유지보수

### 6.1 모듈화
- 기능별로 파일 분리
- 재사용 가능한 컴포넌트 개발

### 6.2 테스트 가능성
- 단위 테스트를 위한 서비스 계층 분리
- 의존성 주입을 통한 테스트 용이성 확보

이 아키텍처 설계는 프로젝트의 기술적 기반을 제공하며, 이후 개발 단계에서 참고되어야 합니다.
